### Generative model (생성 모델)

- 모델에 의해서 데이터가 랜덤하게 제너레이트 되어서 만들어져있다고 보는것

- 결과 데이터를 보고 확률적으로 계산했을때 Total probability of having the observable data가 나옴
- Likelyhood
  - 데이터를 생성했을 확률이 제일 높은것 

- Probabilist Modeling!! => 클러스터링이나 추천 활용



### EM Algorithm

- Maximum Likelyhood 식을 파라미터로 미분해서 0으로 놓아서 찾음

- Cycle이 있어서 단계적으로 계산가능

- random 하게 파라미터 세팅
- converge하게 되면  stop 하고 파라미터 최종값을 찾음
- 역으로 생성 모델을 찾아가는 알고리즘



### PLSI(Probabilistic Latent Semantic Indexing)

- EM Algorithm에서는 데이터가 있으면 클러스터가 바로 결정

- 하지만 PLSI는 내가 문서를 쓴 거를 가지고 모델링을 함

- k개의 주제가 있어서 문서를 주제별로 클러스팅함

  1. 주제를 하나 선택
     - k개의 주제가 있는데, 사람별로 어떤 주제를 쓸지 확률을 구함
     - 모든 주제마다 확률이 다 다르게 책정

  2. 다시 주제를 선택 후 단어 확률 분포 분석

  3. 다시 주제를 선택 후 단어에 대한 확률 분포를 보고 또 random 하게 선택

- 이러한 과정을 거쳐 주제를 선택하고,  그 중에 확률 분포에 의해 단어가 random 하게 하나 선택되고 그게 내가 가지고 있는 제너레이트 모델임



### TWITOBI

- 주제가 선택되면 어떤 단어들이 선택될지의 확률 분포를 분석
- 모델링
  - 사람들마다 팔로우할 확률
  - 유저가 자기 주제를 선택할 확률
  - 팔로우하는 사람드링 주제를 선택할 확률

- 이전에 학습한 내용들을 기반으로 알고리즘 작성



### 느낀점

1강에서 알고리즘에 대해 설명했다면 2강은 좀 더 모델에 대해 설명 해 주신것 같았습니다. 생성 모델 등을 빨간공 파란공을 이용해서 예시를 잘 들어주셔서 이해하기 좋았던것 같습니다. 이 외에도 PLSI와 TWITOBL와 같은 모델링에 대해 배울 수 있는 계기였습니다.
